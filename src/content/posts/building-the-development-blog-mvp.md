오늘은 개인 개발 블로그 MVP를 실제로 구현할 수 있는 상태까지 정리했습니다.

## 설계에서 구현으로

먼저 서비스 기획 문서와 디자인 문서의 용어를 맞추고, 홈·게시글 상세·프로젝트·About·404 화면의 구조를 와이어프레임으로 정리했습니다. 이후 승인된 고해상도 목업을 기준으로 색상, 간격, 타이포그래피, 테마 상태를 구현 기준으로 옮겼습니다.

홈 화면은 게시글이 하나도 없는 초기 상태에서도 완성된 화면처럼 보이도록 구성했습니다. `$ whoami` 형식의 터미널 히어로와 블로그 소개를 먼저 보여주고, 게시글이 없을 때는 카테고리 필터를 숨긴 채 새 글 준비 상태를 안내합니다.

## 구현한 구조

- Astro static output 기반 프로젝트
- Markdown/MDX 콘텐츠 컬렉션
- 게시글과 프로젝트 frontmatter 스키마
- 게시글 정렬과 읽는 시간 계산 유틸리티
- light/dark 테마 저장
- 키보드로 사용할 수 있는 모바일 메뉴
- 프로젝트 목록과 상세 specification sheet
- Categories, Tags, About, 404 라우트

프로젝트 상세는 다음과 같은 정보 구조를 사용합니다.

```text
STATUS
PERIOD
ROLE
STACK
GITHUB (when available)
DEMO (when available)
```

외부 링크는 실제 URL이 있을 때만 렌더링하도록 조건부로 처리했습니다. 아직 확인되지 않은 저장소나 배포 주소는 화면에 만들지 않았습니다.

## 검증한 내용

구현 후 다음 검사를 실행했습니다.

- Astro 타입 진단
- ESLint 정적 분석
- Prettier 형식 검사
- 게시글 정렬·읽는 시간 단위 테스트
- 정적 프로덕션 빌드
- 브라우저에서 홈 빈 상태, 테마 전환, 모바일 메뉴, 프로젝트 상세, 404 확인

특히 360px 모바일 폭에서 메뉴가 열리고 닫히는지, 테마 버튼의 접근 가능한 이름이 상태에 따라 바뀌는지 확인했습니다. 게시글이 추가된 뒤에는 홈에서 카테고리 필터와 최신 글 목록이 자동으로 나타나도록 연결했습니다.

## 다음 단계

다음 단계는 실제 경험과 결과가 확인된 게시글을 같은 콘텐츠 스키마로 추가하는 것입니다. 지금은 오늘 구현한 MVP 자체를 포트폴리오 형태의 작업 기록으로 남겼고, 개인 정보나 검증되지 않은 성과를 임의로 채우지 않는 원칙을 유지했습니다.
