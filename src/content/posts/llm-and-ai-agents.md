LLM과 AI 에이전트는 함께 언급되지만 같은 개념은 아닙니다. LLM이 언어를 처리하는 핵심 모델이라면, AI 에이전트는 모델을 중심으로 목표, 도구, 상태 관리 같은 실행 구조를 더한 시스템입니다.

## LLM은 답변을 생성한다

LLM은 입력된 문맥을 바탕으로 텍스트를 생성합니다. 사용자의 질문에 설명을 제공하거나 코드를 작성하는 작업은 LLM만으로도 수행할 수 있습니다.

하지만 LLM 자체가 외부 시스템의 상태를 바꾸거나 실제 파일을 수정하는 것은 아닙니다. 그런 작업을 하려면 애플리케이션이 별도의 도구와 실행 권한을 연결해야 합니다.

## 에이전트는 목표를 향해 작업한다

AI 에이전트는 다음과 같은 흐름을 가질 수 있습니다.

1. 사용자의 목표를 해석한다.
2. 필요한 정보를 확인한다.
3. 적절한 도구를 선택한다.
4. 도구의 결과를 다시 읽는다.
5. 목표가 달성될 때까지 다음 작업을 결정한다.

예를 들어 문서 검색, 계산, 파일 읽기, API 호출을 도구로 제공하면 에이전트는 답변 생성과 실제 작업을 연결할 수 있습니다.

## 둘의 관계

에이전트가 반드시 LLM만으로 구성되는 것은 아니지만, 자연어 목표를 이해하고 다음 작업을 결정하는 부분에는 LLM이 자주 사용됩니다. 따라서 LLM은 에이전트의 판단을 돕는 두뇌에 가깝고, 도구와 실행기는 실제 행동을 담당한다고 볼 수 있습니다.

```text
AI 에이전트 = LLM + 목표 관리 + 도구 + 실행 결과 확인
```

## 안전하게 설계하려면

도구를 연결할 때는 모델이 할 수 있는 작업의 범위를 명확히 제한해야 합니다. 파일 삭제나 외부 서비스 변경처럼 영향이 큰 작업은 사용자 확인을 거치고, 각 단계의 입력과 결과를 기록하면 문제를 추적하기 쉽습니다.

## 정리

LLM은 언어를 생성하는 모델이고, AI 에이전트는 LLM과 도구를 조합해 목표를 달성하는 실행 시스템입니다. 이 차이를 이해하면 단순한 챗봇과 작업을 수행하는 애플리케이션을 구분해서 설계할 수 있습니다.
