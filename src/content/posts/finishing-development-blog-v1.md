개발 블로그는 기능을 계속 추가할 수 있기 때문에, 어느 시점에 v1을 끝낼지 먼저 정하는 일이 중요합니다. 이번 단계에서는 새 기능의 수보다 실제 글을 안전하게 발행하고 독자가 안정적으로 읽을 수 있는지를 완료 기준으로 삼았습니다.

## v1 종료 범위

v1은 Markdown 콘텐츠를 중심으로 한 정적 블로그입니다. 글을 추가하면 목록, 상세, 카테고리, 태그 페이지가 빌드 과정에서 생성되고, 초안은 공개 결과에서 제외됩니다. 프로젝트와 About 페이지는 확인된 정보만 표시하며, 선택적 외부 링크가 없을 때 빈 링크나 가짜 안내를 만들지 않습니다.

게시글 상세 화면에는 자동 목차, 코드 복사, 이전·다음 글, 관련 글을 포함했습니다. 테마 전환, 모바일 메뉴, 댓글 상태는 브라우저 JavaScript가 필요한 범위에서만 동작하고, 본문과 기본 탐색은 JavaScript 없이도 읽을 수 있도록 유지했습니다.

## 콘텐츠 경계 정리

초기 구현에서 사용하던 날짜와 분류 필드를 v1 규칙에 맞춰 `publishedAt`, `updatedAt`, 기술 주제 카테고리, 콘텐츠 형식으로 정리했습니다. 날짜가 같은 글은 영문 ID를 기준으로 정렬해 빌드마다 결과가 달라지지 않도록 했습니다.

읽기 시간은 본문 문자 수를 기준으로 계산하고, 코드 블록에는 별도의 시간을 더합니다. 이 값은 frontmatter에 직접 입력하지 않고 빌드 시 계산해 글 내용과 표시 값이 어긋나지 않게 했습니다.

## 정적 사이트의 공개 품질

페이지별 canonical URL, Open Graph, Twitter Card 메타데이터를 추가했습니다. 검색 엔진이 정적 경로를 발견할 수 있도록 sitemap과 robots.txt를 생성하고, 공개 글만 RSS 피드에 포함했습니다.

이 구조는 별도 검색 서버나 런타임 데이터베이스 없이도 정적 배포를 유지합니다. 댓글처럼 정적 파일만으로 처리할 수 없는 독자 생성 데이터만 승인된 백엔드 경계를 사용합니다.

## 검증 결과

다음 검사를 실행했습니다.

- `npm run check`
- `npm run lint`
- `npm run format:check`
- `npm run test:unit`
- `npm run build`
- Chromium, Firefox, WebKit에서 댓글·접근성 시나리오 5개

정적 빌드는 RSS, sitemap, robots.txt를 포함해 완료되었고, 세 브라우저의 댓글 로딩·빈 상태·재시도·등록·JavaScript 비활성 상태를 확인했습니다. Firefox는 브라우저 subprocess 권한 문제를 해결한 뒤 다시 실행해 통과시켰습니다.

## 남겨 둔 다음 단계

전체 텍스트 검색, 연·월별 아카이브, 시리즈 페이지, 자동 OG 이미지, 댓글 운영 개선은 v1.1 후보로 남겼습니다. v1 종료는 기능 개발의 끝이 아니라, 운영 중 확인한 문제를 근거로 다음 범위를 정할 수 있는 안정된 기준점입니다.
