개인 개발 블로그는 글을 쓰고 관리하는 흐름이 단순해야 오래 운영할 수 있습니다. 이번 프로젝트는 별도의 서버나 데이터베이스 없이, 로컬 Markdown 파일을 정적 HTML로 만들어 제공하는 방향으로 시작했습니다.

이 글에서는 현재 프로젝트에서 확인된 기술 스택과 각 기술을 선택한 이유를 정리합니다.

## 전체 구성

현재 블로그의 핵심 스택은 다음과 같습니다.

| 기술       | 맡은 역할                                 |
| ---------- | ----------------------------------------- |
| Astro      | 페이지 구성과 정적 사이트 빌드            |
| TypeScript | 콘텐츠 처리와 컴포넌트 로직의 타입 안전성 |
| Markdown   | 게시글과 프로젝트 콘텐츠 작성             |
| CSS        | 반응형 레이아웃과 라이트·다크 테마        |

Astro가 페이지와 빌드 흐름을 담당하고, Markdown 파일은 콘텐츠 저장소 역할을 합니다. TypeScript와 CSS는 각각 코드의 안정성과 화면 표현을 보완합니다.

## Astro: 콘텐츠 중심 사이트에 맞는 프레임워크

이 블로그는 게시글을 읽는 경험이 핵심이므로, 기본 페이지는 브라우저에서 실행되는 JavaScript를 최소화하는 것이 중요합니다. Astro는 컴포넌트 기반으로 화면을 구성하면서 정적 HTML을 생성할 수 있어 이 요구에 잘 맞습니다.

현재 프로젝트는 Astro를 정적 출력으로 사용합니다. 빌드 시점에 게시글과 프로젝트를 읽어 필요한 경로를 만들기 때문에, 방문자가 페이지를 열 때 별도의 API 서버나 데이터베이스를 요청하지 않습니다.

이 구조의 장점은 분명합니다.

- 배포 대상이 정적 파일이라 운영 구조가 단순합니다.
- 콘텐츠가 바뀌지 않으면 런타임 서버가 필요하지 않습니다.
- 글 목록, 카테고리, 태그, 상세 페이지를 하나의 콘텐츠 모델에서 생성할 수 있습니다.

## Markdown: 글과 코드를 함께 관리하기

게시글은 Markdown 파일로 작성하고, frontmatter에 제목·설명·날짜·분류·태그 같은 메타데이터를 기록합니다.

```yaml
title: 정적 개발 블로그의 기술 스택을 선택한 이유
publishedAt: '2026-07-16'
type: Technical Articles
category: Astro
tags:
  - Astro
  - TypeScript
draft: true
```

본문과 메타데이터를 분리하면 글은 일반 문서처럼 작성하면서도 목록과 분류 페이지에 필요한 정보를 일관되게 사용할 수 있습니다. 새 Markdown 파일을 추가하면 같은 페이지 구조를 재사용할 수 있다는 점도 프로젝트 방향과 맞습니다.

## TypeScript: 콘텐츠 경계에서 오류 줄이기

Markdown은 자유롭게 작성할 수 있지만, frontmatter까지 자유로운 문자열로 두면 오타가 빌드 이후에 발견될 수 있습니다. 그래서 콘텐츠 컬렉션 스키마에서 날짜 형식, 글 유형, 카테고리, 태그 배열을 검사합니다.

예를 들어 허용된 카테고리 외의 값을 입력하거나 날짜 형식이 규칙에 맞지 않으면 정적 빌드가 실패합니다. 콘텐츠를 화면에 보여주기 전에 경계에서 검증하기 때문에, 페이지 컴포넌트는 검증된 데이터라는 전제 아래 더 단순하게 유지할 수 있습니다.

## CSS: 작은 화면과 두 가지 테마

화면 스타일은 별도의 UI 프레임워크 없이 CSS로 구성합니다. 디자인 문서에 정의된 색상과 간격을 토큰으로 관리하고, 모바일 우선 규칙으로 데스크톱까지 확장합니다.

특히 다음 상태를 기본 요구사항으로 둡니다.

- 라이트 모드와 다크 모드
- 360px부터 데스크톱까지의 반응형 레이아웃
- 긴 코드와 표의 가로 스크롤
- 키보드 포커스와 200% 확대 시의 가독성

화면을 꾸미는 것보다 글과 코드가 어떤 폭에서도 읽히는 것을 우선했습니다. 정적 사이트의 장점을 유지하기 위해 테마 전환이나 모바일 메뉴처럼 꼭 필요한 상호작용에만 브라우저 JavaScript를 사용합니다.

## 마무리

현재 스택은 Astro를 중심으로 Markdown 콘텐츠, TypeScript 검증, CSS 표현을 조합한 형태입니다. 기술을 많이 추가하기보다 콘텐츠 작성과 로컬 검증이라는 블로그의 핵심 흐름을 짧게 유지하는 데 초점을 맞췄습니다.

앞으로 실제 글을 추가하면서 이 구조가 작성 부담을 줄이는지, 콘텐츠가 늘어났을 때 분류와 탐색이 여전히 편한지를 확인할 예정입니다.
