이번 단계에서는 Astro 정적 블로그의 정보 구조와 콘텐츠 흐름을 유지하면서 화면의 인상을 편집형 비주얼 시스템으로 전환했습니다. Kimi가 만든 디자인 변경을 기준으로 홈 히어로, 공통 레이아웃, 색상 토큰과 페이지 전반의 표현 규칙을 하나의 스타일 계층으로 정리했습니다.

## 기능을 바꾸지 않고 인상을 바꾸는 범위

재설계의 핵심은 새로운 기능을 추가하는 일이 아니었습니다. 기존의 게시물 목록, 카테고리, 프로젝트, 댓글과 정적 라우팅을 그대로 두고 다음 표현 계층을 바꾸는 작업이었습니다.

- 따뜻한 종이색에 가까운 캔버스와 주황 계열 강조색
- 제목에 세리프 글꼴을 사용하는 편집형 타이포그래피
- 선과 넓은 여백을 중심으로 구분하는 레이아웃
- 라이트·다크 테마에 같은 의미를 유지하는 시맨틱 색상 토큰
- 카드, 배지, 탐색 영역, 프로젝트와 글 상세 화면에 공유되는 스타일

이 범위를 별도 `editorial.css`에 모으고 공통 레이아웃에서 한 번 불러오도록 했습니다. 기존 `global.css`의 구조를 삭제하거나 각 Astro 컴포넌트에 같은 규칙을 반복하지 않아, 디자인 계층을 독립적으로 검토할 수 있게 했습니다.

## 색상 값을 의미로 관리하기

밝은 테마와 어두운 테마는 서로 다른 색상표를 사용하지만 컴포넌트가 참조하는 이름은 같습니다. `canvas`, `ink`, `body`, `mute`, `surface`, `hairline`, `accent`처럼 역할을 나타내는 토큰을 유지하고 실제 값만 테마별로 교체했습니다.

이 방식에서는 강조색을 바꾸기 위해 각 버튼과 링크를 찾아 수정할 필요가 없습니다. 컴포넌트는 “주황색”이 아니라 “강조색”을 사용하므로, 테마가 바뀌어도 정보의 우선순위가 유지됩니다. 터미널 영역의 배경·본문·강조색도 같은 방식으로 분리해 일반 화면과 어두운 표면의 대비를 각각 조정했습니다.

## 홈 히어로에서 편집형 리듬 만들기

홈 히어로는 어두운 터미널 패널에서 캔버스와 연결되는 큰 제목 영역으로 바뀌었습니다. 제목 크기는 고정값 대신 `clamp()`를 사용해 화면 폭에 따라 변하도록 했고, 모바일 구간에서는 높이와 글자 크기를 별도로 낮췄습니다.

본문 소개에도 같은 세리프 계열과 넓은 줄 간격을 적용했습니다. 명령 프롬프트와 섹션 번호는 작은 모노스페이스 텍스트와 강조색으로 남겨 블로그의 개발자 정체성을 유지했습니다. 즉, 터미널 문법을 장식 전체로 확대하지 않고 탐색 표식으로 제한했습니다.

## 공통 스타일 계층의 장점과 주의점

하나의 편집형 스타일시트에서 헤더, 목록, 글 상세, 프로젝트, 푸터와 반응형 규칙을 함께 관리하면 화면 간 일관성을 빠르게 확보할 수 있습니다. 반면 선택자 범위가 넓어 기존 컴포넌트 스타일과 우선순위가 충돌할 수 있습니다.

이번 구조에서는 공통 레이아웃에서 `global.css` 다음에 `editorial.css`를 불러와 재설계 규칙이 의도적으로 후순위가 되게 했습니다. 이후 컴포넌트를 수정할 때는 새 규칙을 계속 덧붙이기보다, 공통 표현인지 컴포넌트 고유 표현인지 먼저 구분해야 합니다. 그렇지 않으면 두 스타일 계층이 같은 책임을 나눠 갖게 됩니다.

## 확인한 결과와 남은 검증

변경 후 Astro와 TypeScript 진단, ESLint, 변경 파일의 Prettier 검사, 단위 테스트 8개와 Playwright E2E 테스트 15개가 통과했습니다. E2E는 Chromium, Firefox, WebKit에서 실행됐고 정적 프로덕션 빌드는 44개 페이지를 생성했습니다.

저장소 전체 `format:check`는 이번 변경 파일이 아닌 기존 파일 53개의 서식 불일치 때문에 통과하지 못했습니다. 또한 이번 실행 환경에는 연결 가능한 브라우저 세션이 없어 360px와 1280px 화면의 수동 시각 검토는 완료하지 못했습니다. 따라서 자동화가 보장하지 않는 글자 줄바꿈, 실제 색 대비와 화면별 시각 균형은 후속 브라우저 QA 항목으로 남아 있습니다.
