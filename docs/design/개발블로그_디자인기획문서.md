# 개발 블로그 디자인 기획 문서 (초안)

> 본 문서는 `기획문서.md`(서비스·기능 기획)와 `DESIGN-opencode_ai.md`(레퍼런스 디자인 시스템)를 기반으로, 사용자와의 논의를 거쳐 정리한 디자인 방향 초안이다. 아직 확정본이 아니며, 추가 논의·수정을 거쳐 승인 후 구현 단계로 넘어간다.

---

## 1. 디자인 콘셉트

- **레퍼런스**: OpenCode.ai 마케팅 사이트의 터미널 미학 (Berkeley Mono, 크림/잉크 플랫 컬러, ASCII 브래킷 불릿, 그림자·그라데이션 없는 완전 플랫 시스템)
- **적용 방향**: 원본을 그대로 가져오지 않고 **블로그 맥락에 맞게 변형**
  - 터미널 미학은 유지하되 **살짝 완화** — 채용 담당자·비개발자 협업자도 읽는 사이트이므로 개성은 살리되 진입장벽은 낮춘다
  - 브래킷(`[+]`, `[-]`)·ASCII 요소는 정체성으로 유지
  - 넉넉한 여백(96px 섹션 리듬)은 유지 — "정돈된 느낌"에 기여
  - 컨테이너 radius를 완전히 각지게(`0px`)만 두지 않고, 카드류에 미세한 radius(6~8px 검토)를 줘서 딱딱함 완화하는 방향도 고려

### 미확정 — 후속 논의 필요
- [ ] 카드류 radius 정확한 수치 (0px 유지 vs 6~8px)
- [ ] 브래킷 마커의 본문 내 사용 빈도 가이드라인

## 4. 페이지별 레이아웃

### 4.1 홈

**구성 순서**
1. **다크 터미널 히어로** — `$ whoami` 스타일. 이름 + 짧은 태그라인만 표시 (콘텐츠 데이터에 의존하지 않음 → 글이 0개인 초기 상태에서도 안정적으로 보임)
2. **소개문구** — 히어로와 별도의 텍스트 블록, "AI Agent를 활용해 아이디어를 실제 서비스로 완성하는 풀스택 개발자" 카피를 확장한 설명
3. **카테고리 필터** — `button-tab`/`button-tab-active` 컴포넌트 재활용 (기존 install-method 탭 패턴)
4. **글 목록** — 최신순, 아래 4.2 카드 디자인 참조
5. **빈 상태** — 글이 없을 때 안내 문구 (기획문서 4.1의 문구 그대로 사용)

### 4.2 글 목록 카드

```
[학습메모] AI Agent로 블로그 기획하기
           2026-07-15 · AI Agent · 5분 읽기
           요약 텍스트가 여기 들어가고 길면 말줄임 처리...
```

- 유형 라벨(학습메모/문제해결/기술아티클/프로젝트회고)을 **왼쪽 브래킷 프리픽스**로 배치 — `badge-news` 컴포넌트 재활용
- 3줄 고정 구조: ① 유형+제목, ② 날짜·카테고리·읽기시간, ③ 요약
- 요약은 **항상 노출** (모바일 포함, 숨기지 않음)
- 정렬 순서는 유형과 무관하게 게시일 최신순

### 4.3 글 상세

```
┌───────────┬──────────────────────────────┐
│ [+] 목차   │   제목 / 메타정보               │
│ · 섹션1    │   본문                         │
│ · 섹션2    │   (코드블록, 이미지, 표 등)      │
│   - 소제목 │                                │
│ · 섹션3    │   태그                          │
│           │   이전글 / 다음글                │
│           │   관련 글                       │
└───────────┴──────────────────────────────┘
  ~200px TOC      ~720px 본문
```

- **TOC**: 왼쪽 sticky 사이드바, 항상 표시(헤딩이 적어도 숨기지 않음), 스크롤에 따라 현재 섹션 하이라이트
  - tablet(850px) 이하에서는 사이드바 대신 본문 상단 `[+] 목차` 접이식으로 전환 (`faq-row`의 브래킷 토글 패턴 재활용)
- **본문 폭**: 약 720px (장문 읽기에 적정한 라인 길이로, 기존 960px보다 축소)
- **메타정보**: 1줄 유형 배지+카테고리, 2줄 작성일·수정일·읽기시간. 태그는 본문 하단에 배치
- **코드블록**: `install-snippet` 컴포넌트 확장 — 상단에 언어 라벨 + 복사 아이콘 추가
- **이전글/다음글**: 본문 하단 좌우 2분할 카드
- **관련 글**: 그 아래 목록 아이템 축약형 2~3개

### 4.4 카테고리 / 태그

- 카테고리·태그별 글 개수 표시
- 글이 없는 카테고리/태그는 페이지 자체를 생성하지 않음 (기획문서 4.3)
- 레이아웃은 홈의 글 목록 카드와 동일한 컴포넌트 재사용

### 4.5 프로젝트 목록 / 상세

- **목록**: 프로젝트명, 한 줄 설명, 상태, 담당 역할, 핵심 기술 — 카드형
- **상세**: 정보량이 가장 많은 페이지
  - 상단에 **스펙시트 카드** (`cat project.json` 느낌) — 상태·기간·역할·기술스택을 한눈에 요약
  - 그 아래 섹션별 서술형 콘텐츠(배경, 목표, 역할, 문제해결, QA 결과, 회고 등), 96px 섹션 리듬 유지
  - 값이 없는 링크(repoUrl, demoUrl)는 UI 자체를 숨김 (기획문서 6.2)

### 4.6 소개(About)

- 실명 + 프로필 이미지
- 대표 소개 문구
- 관심 분야·기술 스택 (나열이 아닌 흐름으로 설명)
- 대표 프로젝트·글 링크
- 연락 수단은 이메일 노출 대신 버튼 형태 우선

### 미확정 — 후속 논의 필요
- [ ] 프로젝트 상세 스펙시트 카드의 정확한 필드 배치
- [ ] 소개 페이지 상세 레이아웃 (아직 세부 논의 안 함)
- [ ] 태그/카테고리 페이지 상단에 필터·정렬 옵션 필요 여부

## 7. 접근성

- **포커스 스타일**: 레퍼런스 원칙(halo 없는 flat 포커스)은 키보드 접근성에 불리 → **`:focus-visible`에서만 2px accent 아웃라인** 적용, 마우스 클릭 시엔 기존 flat 유지. 브랜드 톤 유지 + WCAG 충족 절충안
- 명암 대비, 키보드 탐색, 대체 텍스트는 기획문서 8.1 기준 준수 (별도 QA 단계에서 `AGENTS.md` 기준으로 검증)
