# DEV.LOG 기능 구현 현황

> 기준일: 2026-07-17

## 1. 페이지 및 콘텐츠 흐름

| 기능 | 상태 | 현재 구현 내용 |
| --- | --- | --- |
| Home | 완료 | 터미널형 hero, 소개, 최신 글, 카테고리 필터, 빈 상태 |
| Logs | 완료 | 공개 글 전체 목록과 카테고리 필터 제공 |
| 게시글 상세 | 완료 | 본문, 메타데이터, 태그, 목차, 이전·다음 글, 관련 글, 승인형 댓글 UI |
| 카테고리 인덱스 | 완료 | 카테고리별 공개 글 수 표시 |
| 카테고리 상세 | 완료 | 선택한 카테고리의 글을 최신순으로 표시 |
| 태그 인덱스 | 완료 | 글 수 내림차순, 동률 시 이름순으로 태그 표시 |
| 태그 상세 | 완료 | 선택한 태그가 포함된 글 목록 표시 |
| 프로젝트 목록 | 완료 | 프로젝트 요약, 상태, 역할, 기술 스택 표시 |
| 프로젝트 상세 | 부분 완료 | 프로젝트 본문과 JSON 스타일 명세, 관련 프로젝트 글 표시 |
| About | 부분 완료 | 관심 분야, 기술 사용, AI 협업 소개만 구현됨 |
| 404 | 완료 | 오류 안내와 Home 복귀 링크 제공 |

현재 정적 라우트 구조는 다음과 같습니다.

```text
/
/logs/
/posts/{slug}/
/categories/
/categories/{slug}/
/tags/
/tags/{slug}/
/projects/
/projects/{slug}/
/about/
/404.html
```

## 2. 게시글 관리

| 기능 | 상태 | 비고 |
| --- | --- | --- |
| Markdown·MDX 게시글 등록 | 완료 | 파일 추가 시 정적 목록·상세 페이지 생성 |
| frontmatter 스키마 검증 | 완료 | 필수값, enum, 날짜 형식, slug, 태그 중복 검사 |
| 초안 제외 | 완료 | `draft: true` 게시글을 공개 목록과 상세 경로에서 제외 |
| 최신순 정렬 | 완료 | 날짜 내림차순, 같은 날짜는 ID 오름차순 |
| 읽기 시간 | 부분 완료 | 현재는 220 words/minute 방식이며 한국어 500자 및 코드 블록 가산 설계와 다름 |
| 콘텐츠 유형 | 완료 | Article, Tutorial, Note, Retrospective |
| 기술 카테고리 | 완료 | AI, Frontend, Backend, Projects, Devlog |
| 카테고리 브라우저 필터 | 완료 | 필터 결과 0건 안내와 전체 글 복귀 문구 포함 |

## 3. 게시글 상세 기능

| 기능 | 상태 | 비고 |
| --- | --- | --- |
| H2·H3 자동 목차 | 완료 | Markdown heading에서 항목 생성 |
| 현재 목차 항목 강조 | 완료 | 스크롤 위치에 따라 강조 및 `aria-current` 적용 |
| 데스크톱 sticky 목차 | 완료 | 약 200px 보조 열에 고정 |

| heading anchor 이동 | 완료 | 문장부호를 제거한 heading ID 규칙 적용 |
| 코드 구문 강조 | 완료 | Astro Markdown 렌더링의 코드 출력을 사용 |
| 코드 언어 표시 | 완료 | 코드 class에서 언어명을 읽어 표시 |
| 코드 복사 | 완료 | 성공·실패 문구와 2초 후 초기화, live status 제공 |
| 태그 링크 | 완료 | 각 태그에서 태그 상세 화면으로 이동 |
| 이전·다음 글 | 완료 | 같은 콘텐츠 유형 안에서 날짜순 이동 |
| 관련 글 | 완료 | 공유 태그 우선, 같은 카테고리 보완, 최대 3개 |

## 5. 댓글 백엔드와 프론트 연결

| 기능 | 상태 | 비고 |
| --- | --- | --- |
| 댓글 DB 스키마 | 개발 완료 | 개발 Supabase에 댓글 상태, 인덱스, 요청 제한 테이블과 원자적 RPC 적용·검증 |
| 승인 댓글 공개 조회 | 개발 완료 | 실제 Edge Function GET과 RLS에서 `approved` 상태만 반환하는 흐름 검증 |
| 댓글 등록 | 개발 완료 | 실제 POST에서 입력 검증, honeypot, HMAC 요청 제한 후 `pending` 저장 검증 |
| 댓글 UI | 완료 | 로딩, 빈 목록, 목록, 등록 성공, API 오류, 재시도 상태 제공 |
| 기능 플래그 | 완료 | 기본값은 비활성화이며 빌드 환경 변수가 있을 때만 댓글 UI와 요청 활성화 |
| 외부 Supabase 연결 | 개발 완료·운영 대기 | 개발 프로젝트 적용과 실동작 검증 완료, production 및 실제 사이트 Origin 연결은 미수행 |

## 6. 프로젝트 및 프로필 기능

| 기능 | 상태 | 비고 |
| --- | --- | --- |
| 프로젝트 컬렉션 | 완료 | 로컬 Markdown과 스키마 사용 |
| 프로젝트 목록·상세 경로 | 완료 | 파일 기반 정적 페이지 생성 |
| 프로젝트 상태·기간·역할·스택 | 부분 완료 | 기본 정보는 보이나 종료일이 없을 때 임시 날짜 문자열이 표시됨 |
| 저장소·데모 링크 | 부분 완료 | 스키마는 지원하지만 상세 화면에 표시하는 UI가 없음 |

| 실제 프로필 정보 | 부분 완료 | GitHub URL은 있으나 이름, 프로필 이미지, 대표 글·프로젝트 영역은 없음 |
| 연락 기능 | 미구현 | 현재 MVP 범위에서 의도적으로 제외 |

## 7. 품질 및 배포 상태

| 기능 | 상태 | 비고 |
| --- | --- | --- |
| TypeScript·Astro 검사 | 설정됨 | `npm run check` |
| 정적 분석 | 설정됨 | `npm run lint` |
| 포맷 검사 | 설정됨 | `npm run format:check` |
| 단위 테스트 | 부분 완료 | 기존 유틸리티와 댓글 입력 경계값을 합쳐 8개 시나리오 검사 |
| E2E 테스트 | 부분 완료 | 댓글 핵심 흐름 5개를 3종 브라우저에서 검사, 전체 사이트 회귀는 남음 |
| 자동 접근성 테스트 | 부분 완료 | 댓글 영역 axe 검사는 추가됐지만 전체 주요 화면 검사는 남음 |
| 브라우저 수동 QA | 부분 완료 | Chromium 기반 TOC QA 문서는 있으나 최신 전체 화면 회귀 검증은 아님 |
| GitHub Pages 자동 배포 | 설정됨 | main push 및 수동 실행 워크플로 존재 |
| 실제 라이브 배포 상태 | 검증 필요 | 이번 현황 작성에서는 원격 상태를 확인하지 않음 |
