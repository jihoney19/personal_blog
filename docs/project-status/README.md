# DEV.LOG 프로젝트 현황 요약

> 기준일: 2026-07-17
> 기준: 현재 저장소의 코드, 설정, 콘텐츠, 테스트, 디자인 문서

이 디렉터리는 DEV.LOG 프로젝트가 현재 어떤 기술로 만들어졌고, 어떤 기능까지 구현되었으며, 어떤 디자인 원칙과 부가 기능을 갖고 있는지 빠르게 파악하기 위한 현황 문서입니다.

## 현재 완성도 판단

현재 프로젝트는 콘텐츠를 읽고 탐색하는 정적 블로그의 기본 흐름 위에 **첫 운영 기능인 승인형 댓글의 개발 백엔드까지 배포·검증한 확장 단계**입니다. 글 본문은 계속 Markdown과 Git으로 관리하고, 독자가 만드는 댓글 데이터만 Supabase Postgres와 Edge Function으로 분리했습니다.

다만 다음 항목은 아직 완료로 보기 어렵습니다.

- 모바일 목차가 설계 문서의 접기·펼치기 형태로 구현되지 않음
- About 화면이 소개 문장 중심의 최소 구성으로, 프로필·대표 콘텐츠 구성이 미완성
- 프로젝트 외부 링크 필드는 스키마에 있지만 상세 화면 표시가 없음
- production Supabase와 실제 사이트 빌드 변수·Origin 연결은 아직 수행되지 않음
- SEO는 제목과 설명 메타데이터 수준이며 canonical, Open Graph, RSS 등은 미구현
- Figma 고해상도 화면 동기화는 기존 디자인 문서에서 대기 상태로 기록됨

## 현황 작성 시 검증 결과

| 검증 | 결과 | 세부 내용 |
| --- | --- | --- |
| `npm run qa` | 통과 | Astro·TypeScript 0 errors, ESLint·Prettier 통과, 단위 8개·E2E 15개·접근성 3개 통과, 최종 정적 빌드 통과 |
| 정적 페이지 생성 | 통과 | 게시글 6개를 포함해 총 33페이지 생성 |
| `npm run test:e2e` | 통과 | Chromium, Firefox, WebKit에서 댓글 조회·등록·빈 상태·오류 재시도·JS 비활성 fallback·axe 검사 15개 통과 |
| Supabase 개발 백엔드 | 통과 | `xnttxukulsbqyapzirkw`에 스키마·RLS·RPC·Edge Function·비밀값 적용, 실제 GET·POST·승인·CORS·rate-limit 검증 통과 |
| `npm audit --omit=dev` | 주의 필요 | 기존 Astro 5 계열에서 high 1개·low 2개 보고, 수정 버전은 Astro 메이저 업그레이드 필요 |
| 문서 내부 상대 링크 | 통과 | 이 디렉터리의 문서 링크 대상이 모두 존재함 |

이 문서는 구현 완료를 선언하는 문서가 아니라, 현재 저장소의 스냅샷을 설명하는 문서입니다. 이후 코드나 콘텐츠가 변경되면 함께 갱신해야 합니다.
