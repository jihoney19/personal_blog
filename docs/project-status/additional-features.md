# DEV.LOG 부가 기능 정리

> 기준일: 2026-07-17

이 문서는 페이지의 기본 표시 기능 외에 사용자 편의성, 접근성, 안정성, 운영을 위해 추가된 기능을 정리합니다.

## 1. 사용자 편의 기능

### 테마 자동 선택과 저장

- 첫 방문에는 운영체제의 라이트·다크 선호 설정을 따릅니다.
- 사용자가 선택한 테마는 `localStorage`에 저장됩니다.
- 저장소 접근이 차단된 환경도 예외로 처리합니다.
- head에서 렌더 전에 테마를 적용해 초기 색상 깜빡임을 줄입니다.
- 버튼에 현재 모드와 전환 동작을 설명하는 label과 title을 갱신합니다.

### 카테고리 즉시 필터

- 페이지를 다시 불러오지 않고 게시글을 카테고리별로 필터링합니다.
- 선택 상태를 `aria-pressed`로 전달합니다.
- 결과가 없으면 빈 화면 대신 안내 문구를 표시합니다.

### 목차 추적

- 게시글의 H2·H3에서 목차를 자동 생성합니다.
- 스크롤 위치에 따라 현재 읽는 섹션을 강조합니다.
- URL fragment와 실제 Markdown heading ID가 일치하도록 별도 slug 규칙을 사용합니다.

### 코드 복사

- Markdown 코드 블록마다 언어명과 Copy 버튼을 동적으로 추가합니다.
- 성공 시 `Copied`, 실패 시 재시도를 안내하는 문구를 표시합니다.
- 결과는 화면과 보조 기술에 함께 전달됩니다.

### 승인형 댓글

- 공개 댓글 조회와 새 댓글 등록은 Supabase Edge Function 하나로 통제합니다.
- 새 댓글은 즉시 노출하지 않고 `pending` 상태로 저장해 관리자가 검토합니다.
- honeypot과 10분당 5회 요청 제한으로 기본적인 자동 스팸 제출을 줄입니다.
- 요청 제한 식별자는 원본 IP를 저장하지 않고 서버 비밀값으로 HMAC 처리합니다.
- API가 실패하면 오류와 재시도 버튼을 표시하고, 설정 전에는 기능 플래그로 UI 자체를 숨깁니다.

## 2. 탐색 보조 기능

- 게시글 태그를 클릭하면 같은 태그의 글 목록으로 이동합니다.
- 카테고리·태그 상세 화면에 breadcrumb와 전체 목록 복귀 링크가 있습니다.
- 이전·다음 글은 같은 콘텐츠 유형 안에서 연결합니다.
- 관련 글은 공유 태그를 우선하고 같은 카테고리 글로 보완합니다.
- 데스크톱에서는 고정 사이드바로 주요 화면에 빠르게 이동합니다.
- 모바일 메뉴는 `Esc` 키로 닫을 수 있고 메뉴 버튼으로 포커스가 돌아옵니다.
- 404 화면에서 Home으로 복귀할 수 있습니다.

## 3. 콘텐츠 안정성 기능

- 공개 목록과 상세 경로에서 draft 게시글을 제외합니다.
- 날짜가 같은 게시글은 영문 ID 오름차순으로 정렬해 빌드 결과를 일정하게 유지합니다.
- 태그의 앞뒤 공백과 중복을 스키마에서 검사합니다.
- 프로젝트 종료일이 시작일보다 빠르면 스키마 검증에 실패합니다.
- 외부 링크 값은 URL 형식으로 검증합니다.
- GitHub Pages의 subpath에서도 내부 링크가 맞도록 `sitePath()`로 base path를 보정합니다.

## 4. 반응형·접근성 보조 기능

- 키보드 사용자를 위한 본문 바로가기 링크가 있습니다.
- 필터, 메뉴, 테마, 복사 상태에 ARIA 속성을 적용합니다.
- `:focus-visible`에서 명확한 포커스 표시를 제공합니다.
- 라이트·다크 양쪽에 별도 상태 색상을 둡니다.
- 모바일에서 표와 코드는 해당 요소 내부에서 가로 스크롤됩니다.
- 긴 제목, 메타데이터, 태그가 좁은 화면에서 줄바꿈됩니다.
- `prefers-reduced-motion` 사용자의 불필요한 전환 효과를 줄입니다.
- 외부 GitHub 링크는 새 탭과 `rel="noreferrer"`를 사용합니다.

## 5. 개발·운영 보조 기능

### 로컬 품질 명령

```bash
npm run check
npm run lint
npm run format:check
npm run test:unit
npm run test:e2e
npm run test:a11y
npm run build
npm run qa
```

`npm run qa`는 정적 검사, 포맷 검사, 단위 테스트, 3종 브라우저 E2E, axe 접근성 검사, 최종 프로덕션 빌드를 묶어서 실행합니다.

### 배포 자동화 설정

- main 브랜치 push 또는 수동 실행으로 GitHub Pages 빌드·배포 가능
- Node.js 20과 `npm ci`를 사용하는 재현 가능한 빌드 절차
- GitHub Pages artifact 업로드와 Pages 배포 단계 구성
- Astro base path를 GitHub Actions 환경에서 자동 전환
- 저장소 변수로 댓글 기능 플래그와 공개 Supabase 설정을 빌드에 주입
- Sites 프로젝트 연결을 위한 `.openai/hosting.json` 메타데이터 존재

이번 현황 작성에서는 실제 원격 워크플로 실행 결과와 라이브 사이트 상태를 확인하지 않았습니다.

### 게시 운영 절차

- `BLOG_POSTING.md`에 원격 push 전에 수행해야 할 게시글 선택, 작성, 승인, 공개 절차가 정의되어 있습니다.
- `docs/operations/blog-publication-ledger.md`는 같은 주제와 단계를 중복 게시하지 않기 위한 기록 파일입니다.
- 현재 ledger에는 등록된 게시 이력이 없습니다.

## 6. 아직 추가되지 않은 부가 기능

- 사이트 내 전체 텍스트 검색
- 좋아요 등 댓글 외 사용자 반응
- RSS 공개 URL 검증
- 자동 OG 이미지
- 방문자 분석
- 연·월별 아카이브
- 자동화된 전체 내부 링크·콘솔 오류 E2E 검사
- 댓글 외 주요 화면의 Chromium, Firefox, WebKit 전체 회귀 테스트
- Lighthouse 자동 성능 검사
