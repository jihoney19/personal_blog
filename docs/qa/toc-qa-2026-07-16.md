## 발견된 문제 (수정 전)

콜론(`:`)을 포함한 제목의 TOC 링크 fragment가 실제 Markdown 렌더링 제목 ID와 일치하지 않았습니다. `전체 구성`과 `마무리`는 정상 이동했지만, 콜론이 있는 네 개 항목은 대상 요소가 없어 이동하지 않았습니다.

| TOC 항목 | 수정 전 fragment | 실제 제목 ID | 결과 |
| --- | --- | --- | --- |
| Astro: 콘텐츠 중심 사이트에 맞는 프레임워크 | `#astro:-콘텐츠-중심-사이트에-맞는-프레임워크` | `astro-콘텐츠-중심-사이트에-맞는-프레임워크` | 실패 |
| Markdown: 글과 코드를 함께 관리하기 | `#markdown:-글과-코드를-함께-관리하기` | `markdown-글과-코드를-함께-관리하기` | 실패 |
| TypeScript: 콘텐츠 경계에서 오류 줄이기 | `#typescript:-콘텐츠-경계에서-오류-줄이기` | `typescript-콘텐츠-경계에서-오류-줄이기` | 실패 |
| CSS: 작은 화면과 두 가지 테마 | `#css:-작은-화면과-두-가지-테마` | `css-작은-화면과-두-가지-테마` | 실패 |

## 원인

서버에서 TOC 링크를 만들 때 사용하던 `toSlug()`는 공백만 하이픈으로 바꾸므로 콜론을 남겼습니다. 반면 Markdown 렌더러는 제목 ID를 만들 때 콜론을 제거합니다. 서버 TOC 규칙과 렌더된 제목 ID 규칙이 달라 fragment 대상이 불일치했습니다.

## 수정

- `src/lib/slug.ts`에 Markdown 제목 ID 용도인 `toHeadingId()`를 추가했습니다.
- 일반 목록 URL에 쓰이는 기존 `toSlug()`는 변경하지 않았습니다.
- 게시글 TOC 생성에만 `toHeadingId()`를 적용했습니다.
- 콜론이 포함된 제목의 ID 규칙을 검증하는 단위 테스트를 추가했습니다.

## 재검증 결과

- TOC 6개 링크가 모두 실제 제목 요소를 가리킴을 브라우저 DOM에서 확인했습니다.
- `#astro-콘텐츠-중심-사이트에-맞는-프레임워크` 딥링크 진입 시 대상 제목이 존재하고 해당 위치까지 스크롤되는 것을 확인했습니다. (`scrollY: 1094`, 대상 위치: 약 `0px`)
- `npm run test:unit`, `npm run check`, `npm run lint`, `npm run format:check`, `npm run build`를 통과했습니다.
- `npm run test:e2e`는 저장소에 Playwright 테스트 파일이 없어 `No tests found`로 실행되지 않았습니다.
