# Personal Development Blog 사용자 관점 QA

## 0. QA 정보

| 항목 | 내용 |
|---|---|
| 검토일 | 2026-07-15 |
| 검토 방식 | 실제 미리보기에서 사용자의 탐색 흐름으로 확인 |
| 대상 | Home, Categories, Tags, Post Detail |
| 주요 관점 | 클릭 수, 다음 행동의 명확성, 빈 상태, 링크 연결, 모바일 사용성 |

이 문서는 개발자가 구현한 방식이 아니라 처음 방문한 사용자가 콘텐츠를 찾고 읽는 과정에서 느낄 수 있는 불편을 기록합니다.

## 1. 요약

초기 검토에서 분류 페이지의 중간 클릭, 빈 필터 결과 안내 부재, 목차 anchor 오류가 가장 큰 마찰로 확인되었습니다. 아래 개선안을 구현하고 브라우저에서 재검증했습니다.

## 2. 발견사항

### UQ-01 — 카테고리에서 게시글까지 불필요한 중간 클릭이 필요함

- 우선순위: P1
- 경로: `Home → Categories → Astro → 게시글`
- 현재 동작: Categories에서 `Astro`를 한 번 클릭해 `/categories/astro/`로 이동한 뒤, 게시글 제목을 다시 클릭해야 상세 페이지가 열립니다.
- 사용자 불편: 카테고리 이름을 눌렀을 때 게시글 목록이 이미 보이는데, 게시글을 읽기 위해 같은 목록에서 한 번 더 선택해야 합니다. 게시글이 1개뿐인 경우 특히 불필요하게 느껴집니다.
- 개선 방향:
  - 카테고리 인덱스에 게시글 미리보기 1~3개를 바로 노출합니다.
  - 게시글이 1개인 카테고리는 카테고리 링크 대신 게시글 상세로 바로 연결하거나, 카테고리 상세에서 첫 게시글을 명확한 primary action으로 둡니다.
  - 카테고리 상세의 제목과 게시글 목록을 한 화면에서 이어지는 탐색 흐름으로 시각적으로 묶습니다.

### UQ-02 — 결과가 없는 필터를 선택해도 안내가 없음

- 우선순위: P1
- 경로: `Home → 최신 글 → Backend`
- 현재 동작: Backend를 선택하면 게시글이 숨겨지지만 `검색 결과 없음` 또는 `해당 카테고리의 글이 없습니다` 메시지가 나타나지 않습니다.
- 사용자 불편: 화면이 비어 보이기 때문에 필터가 고장 났는지, 로딩 중인지, 정말 글이 없는지 판단할 수 없습니다.
- 개선 방향:
  - 결과가 0개이면 필터 목록 아래에 빈 상태를 표시합니다.
  - 예: `[+] Backend 글이 없습니다` / `전체 글 보기`
  - 빈 상태에 `전체`로 돌아가는 명확한 action을 제공합니다.

### UQ-03 — 게시글 목차의 `본문` 링크가 실제 위치로 이동하지 않음

- 우선순위: P1
- 경로: `게시글 상세 → Contents → 본문`
- 현재 동작: 링크 href가 `#conclusion`이지만 페이지에 `id="conclusion"`인 요소가 없습니다.
- 사용자 불편: 링크를 눌러도 화면이 이동하지 않아 목차 기능이 고장 난 것처럼 느껴집니다.
- 개선 방향:
  - 실제 본문 마지막 또는 결론 heading에 `id="conclusion"`을 부여합니다.
  - Markdown heading에서 생성한 id를 기준으로 TOC를 자동 생성해 링크와 본문이 항상 일치하게 합니다.

### UQ-04 — 게시글 상세의 태그가 읽기 전용으로만 보임

- 우선순위: P2
- 경로: `게시글 상세 → 태그`
- 현재 동작: `#Astro`, `#Accessibility` 등이 시각적으로 표시되지만 클릭할 수 없습니다.
- 사용자 불편: 관심 있는 태그의 다른 글을 탐색하려면 상단 Tags 메뉴로 이동해 태그를 다시 찾아야 합니다.
- 개선 방향:
  - 상세 페이지의 태그를 `/tags/{slug}/` 링크로 만듭니다.
  - 링크임을 알 수 있도록 hover/focus 상태를 추가합니다.

### UQ-05 — 분류 상세 페이지에서 현재 위치와 돌아가기 경로가 약함

- 우선순위: P2
- 경로: `Categories → Astro`
- 현재 동작: 본문에는 `Astro` 제목만 있고, `Categories로 돌아가기` 또는 breadcrumb가 없습니다.
- 사용자 불편: 브라우저 뒤로가기를 사용하지 않으면 분류 목록으로 돌아가는 명확한 방법이 없습니다.
- 개선 방향:
  - 제목 위에 `Categories / Astro` breadcrumb를 제공합니다.
  - 제목 아래에 `← 모든 카테고리` 링크를 둡니다.
  - Tags 상세에도 동일한 규칙을 적용합니다.

### UQ-06 — 아이콘 전용 테마 버튼의 기능을 처음 보는 사용자가 알기 어려움

- 우선순위: P2
- 경로: 모든 페이지 상단
- 현재 동작: 테마 버튼은 달/해 아이콘만 표시하고 accessible name은 제공하지만, 마우스 사용자를 위한 시각적 설명이나 tooltip은 없습니다.
- 사용자 불편: 아이콘이 테마 전환인지 장식인지 즉시 판단하기 어렵습니다.
- 개선 방향:
  - `title="다크 모드로 전환"`처럼 짧은 native tooltip을 추가합니다.
  - 첫 방문에는 버튼 주변에 한 번만 보이는 짧은 안내를 고려합니다.
  - 아이콘 상태와 현재 테마의 관계를 일관되게 유지합니다.

## 3. 사용성 우선순위

1. UQ-01: 분류에서 게시글까지의 불필요한 중간 단계 축소
2. UQ-02: 필터 결과 없음 상태 추가
3. UQ-03: 깨진 목차 anchor 수정
4. UQ-04: 상세 태그를 탐색 링크로 변경
5. UQ-05: breadcrumb와 돌아가기 링크 추가
6. UQ-06: 아이콘 버튼의 시각적 설명 보강

## 4. 확인한 정상 동작

- 홈에서 최신 게시글과 컴팩트 카테고리 필터가 표시됨
- `Backend` 선택 시 기존 게시글이 숨겨짐
- Astro 카테고리 상세에서 게시글 목록이 열림
- Astro 태그 상세에서 게시글 목록이 열림
- 카테고리·태그 상세의 게시글 제목에서 게시글 상세로 이동 가능
- 모바일 메뉴와 테마 전환 컨트롤이 존재함

## 5. 다음 QA 라운드 제안

- UQ-01~03을 수정한 뒤 동일한 경로를 다시 측정합니다.
- 게시글이 3개 이상일 때 카테고리·태그 탐색이 여전히 자연스러운지 확인합니다.
- 360px에서 분류 목록, 필터, 태그가 줄바꿈되더라도 primary action이 먼저 보이는지 확인합니다.
- 키보드만으로 `필터 → 게시글 → 태그 → 원래 분류`를 왕복할 수 있는지 확인합니다.
